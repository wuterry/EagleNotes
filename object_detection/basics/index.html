    ROIPooling v.s. ROIAlign#

    ROI Pooling 与普通 Pooling 的区别？ ROI Align 与 ROI Pooling 的实现上有什么区别？

    原文来源: Part I Part II

    ROI Pooling#

    一句话总结: 两次量化操作导致目标位置偏移。

    一次量化#

    如上图, ROI 映射后的精确坐标应该是红框所示的浮点型坐标位置, 但是经过量化后只能对齐到整数网格上, 丢失了蓝色区域信息, 增加了绿色区域信息, 这样会导致偏差错位。

    二次量化#

    由于 ROI Pooling 之后接全连接层, 因此需要将不同尺寸的 ROI 区域 Pooling 成相同大小, 如下图量化会导致最下方整行信息丢失。

    ROI Align#

    ROI Align 不再使用量化进行数据池化, 而是将原始的浮点坐标划分成最终等长和等宽的格子, 然后在每个格子内进行 双线性插值。

    假如使用 \(3 \times 3\) Pooling Layer, 则需要将目标区域分割为 \(1.51 \times 2.08\) 大小的格子, 如图

    • 每一个格子都横跨 6 个不同的网格单元, 通过等间隔将该格子划分成 \(3\times3\) 区域, 获取四个采样点位置;
    • 采用双线性插值分别计算格子内采样点的特征值;
    • 使用 Max Pooling 或者 Avg Pooling 池化对应的格子。

    PrROI Pooling#

    2018 ECCV Acquisition of Localization Confidence for Accurate Object Detection

    PrRoI Pooling (Precision ROI Pooling) 解决了 RoI Align 中 N 的取值难以自适应的问题。具体而言，通过双线性插值将离散特征图转换为连续，然后利用二重积分进行池化，提高了目标检测的定位精度。相比于RoI Align，PrRoI Pooling没有划分子区域，而是直接计算平均值，解决了N的取值适应性问题，并且该池化操作可求偏导数。

    IoU 系列#

    IoU 相关计算#

    \[ \begin{aligned} IoU &= \frac{|A \cap B|}{ |A\cup B| } \\ GIoU &= IoU - \frac{|A_c - U|}{|A_c|} \\ DIoU &= \frac{\rho^2 (A, B)}{c^2} \\ CIoU &= DIoU + \alpha v \end{aligned} \]

    IoU 相关 Loss 对比#

    Loss 优点 缺点
    IoU 具有尺度不变性，满足非负性、同一性、对称性、三角不等性等特点 - 如果两个框不相交，无法反映两个框距离远近; - 无法精确的反映两个框的重合度大小
    GIoU 基于IOU引入最小外接框解决检测框和真实框无重叠时loss=0的问题 1.当检测框和真实框出现包含现象的时候GIOU退化成IOU; 2.两个框相交时，在水平和垂直方向上收敛慢
    DIoU 考虑了 GIOU 特点，直接回归两个框中心点的欧式距离，加速收敛 回归过程中未考虑 bbox 的长宽比，精确度待提升
    CIoU DIOU 基础上增加了 bbox 尺度和长宽 loss 1. 长宽比描述的是相对值，具有模糊性； 2. 未考虑难易样本平衡问题
    EIoU CIOU基础上分别计算宽高的差异值取代了长宽比，引入Focal Loss解决难易样本不平衡的问题

    def box_area(boxes: Tensor) -> Tensor:
    """
    Computes the area of a set of bounding boxes, which are specified by their
    (x1, y1, x2, y2) coordinates.

    Args:
        boxes (Tensor[N, 4]): boxes for which the area will be computed. They
            are expected to be in (x1, y1, x2, y2) format with
            ``0 <= x1 < x2`` and ``0 <= y1 < y2``.

    Returns:
        Tensor[N]: the area for each box
    """
    boxes = _upcast(boxes)
    return (boxes[:, 2] - boxes[:, 0]) * (boxes[:, 3] - boxes[:, 1])

def _box_inter_union(boxes1: Tensor, boxes2: Tensor) -> Tuple[Tensor, Tensor]:
    area1 = box_area(boxes1)
    area2 = box_area(boxes2)

    lt = torch.max(boxes1[:, None, :2], boxes2[:, :2])  # [N,M,2]
    rb = torch.min(boxes1[:, None, 2:], boxes2[:, 2:])  # [N,M,2]

    wh = _upcast(rb - lt).clamp(min=0)  # [N,M,2]
    inter = wh[:, :, 0] * wh[:, :, 1]  # [N,M]

    union = area1[:, None] + area2 - inter

    return inter, union

def box_iou(boxes1: Tensor, boxes2: Tensor) -> Tensor:
    """
    Return intersection-over-union (Jaccard index) between two sets of boxes.

    Both sets of boxes are expected to be in ``(x1, y1, x2, y2)`` format with
    ``0 <= x1 < x2`` and ``0 <= y1 < y2``.

    Args:
        boxes1 (Tensor[N, 4]): first set of boxes
        boxes2 (Tensor[M, 4]): second set of boxes

    Returns:
        Tensor[N, M]: the NxM matrix containing the pairwise IoU values for every element in boxes1 and boxes2
    """
    inter, union = _box_inter_union(boxes1, boxes2)
    iou = inter / union
    return iou

    def generalized_box_iou(boxes1: Tensor, boxes2: Tensor) -> Tensor:
    """
    Return generalized intersection-over-union (Jaccard index) between two sets of boxes.

    Both sets of boxes are expected to be in ``(x1, y1, x2, y2)`` format with
    ``0 <= x1 < x2`` and ``0 <= y1 < y2``.

    Args:
        boxes1 (Tensor[N, 4]): first set of boxes
        boxes2 (Tensor[M, 4]): second set of boxes

    Returns:
        Tensor[N, M]: the NxM matrix containing the pairwise generalized IoU values
        for every element in boxes1 and boxes2
    """

    # degenerate boxes gives inf / nan results
    # so do an early check
    assert (boxes1[:, 2:] >= boxes1[:, :2]).all()
    assert (boxes2[:, 2:] >= boxes2[:, :2]).all()

    inter, union = _box_inter_union(boxes1, boxes2)
    iou = inter / union

    lti = torch.min(boxes1[:, None, :2], boxes2[:, :2])
    rbi = torch.max(boxes1[:, None, 2:], boxes2[:, 2:])

    whi = _upcast(rbi - lti).clamp(min=0)  # [N,M,2]
    areai = whi[:, :, 0] * whi[:, :, 1]

    return iou - (areai - union) / areai

    IoU-based Loss#

    IoU Loss#

    优点: 具有尺度不变性.

    缺点: (1) 若两个框无交集, \(IoU = 0\), 此时 \(loss = 0\), 无梯度回传, 不能进行学习; (2) IoU 无法准确反应 GT 与预测框的重合度。

    GIoU Loss#

    动机: 如图, 相同的回归损失 (L2 Loss) 情况下, IoU 值是不一样的。
    L2 Loss 与对应 IoU, GIoU 的变化关系 GIoU 计算示意

    对于任意的框 A, B, 首先找到能够包围住他们的最小框 C, 然后计算 \(C \(A \cup B)\) 的面积与 C 面积的比值。

    性质:

    • GIoU 是 IoU 的下限, 即 \(\text{GIoU}(A, B) \leq \text{IoU}(A, B)\), 且 \(\lim _{A\rightarrow B} \text{GIoU}(A, B) = \text{IoU}(A, B)\);
    • \(-1 \leq \text{GIoU}(A, B) \leq 1\),
      • \(A\), \(B\) 完全重合时, \(\text{GIoU}(A, B) = \text{IoU}(A, B)=1\);
      • \(\frac{A \cup B}{C} \rightarrow 0\), \(\text{GIoU} \rightarrow -1\).

    NMS 系列#

    NMS 对 IOU 大于阈值的检测框进行删除, Soft-NMS 则是降权原有检测框的置信度, 对于有重叠的框，重叠区域越大则置信度衰减越严重。

    """https://github.com/rbgirshick/fast-rcnn/blob/master/lib/utils/nms.py"""

import numpy as np

def nms(dets, thresh):
    x1 = dets[:, 0]
    y1 = dets[:, 1]
    x2 = dets[:, 2]
    y2 = dets[:, 3]
    scores = dets[:, 4]

    areas = (x2 - x1 + 1) * (y2 - y1 + 1)
    order = scores.argsort()[::-1]

    keep = []
    while order.size > 0:
        i = order[0]
        keep.append(i)

        # compute intersection between the highest score bbox with the rest bboxes
        x1_max = np.maximum(x1[i], x1[order[1:]])
        y1_max = np.maximum(y1[i], y1[order[1:]])
        x2_min = np.minimum(x2[i], x2[order[1:]])
        y2_min = np.minimum(y2[i], y2[order[1:]])
        w = np.maximum(0.0, x2_min - x1_max + 1)
        h = np.maximum(0.0, y2_min - y1_max + 1)
        inter = w * h

        # compute ious
        overlap = inter / (areas[i] + areas[order[1:]] - inter)

        inds = np.where(overlap <= thresh)[0]
        order = order[inds + 1]

    return keep

    def soft_nms(dets, sigma=0.5, threshold1=0.7, threshold2=0.1, method=1):
    n = dets.shape[0]

    x1 = dets[:, 0]
    y1 = dets[:, 1]
    x2 = dets[:, 2]
    y2 = dets[:, 3]
    scores = dets[:, 4]
    areas = (y2 - y1 + 1) * (x2 - x1 + 1)

    new_scores = scores.copy()
    index = [i for i in range(n)]
    keep = []

    while len(index) > 0:
        # get max box position of current based new scores
        max_score = 0
        max_pos = -1

        for i in index:
            if new_scores[i] >= max_score:
                max_pos = i
                max_score = new_scores[i]

        if max_pos == -1:
            break

        keep.append(max_pos)
        index.remove(max_pos)

        # calculate ious between current max box and others
        x11 = np.maximum(x1[max_pos], x1[index])
        y11 = np.maximum(y1[max_pos], y1[index])
        x22 = np.minimum(x2[max_pos], x2[index])
        y22 = np.minimum(y2[max_pos], y2[index])

        w = np.maximum(0, x22 - x11 + 1)
        h = np.maximum(0, y22 - y11 + 1)

        overlaps = w * h
        ious = overlaps / (areas[max_pos] + areas[index] - overlaps)

        # adjust score of others
        new_index = []
        for i, ids in enumerate(index):
            iou = ious[i]
            weight = 1

            if method == 1:
                # linear
                if iou >= threshold1:
                    weight = 1 - iou
            elif method == 2:
                # gaussian
                weight = np.exp(-(iou * iou) / sigma)
            else:
                # normal nms
                if iou >= threshold1:
                    weight = 0

            new_scores[ids] = new_scores[ids] * weight

            if new_scores[ids] > threshold2:
                new_index.append(ids)
        index = new_index

    return keep
