    区间问题#

    一般思路: 先排序，后贪心。

    区间合并#

    题意#

    给定一组区间的集合, \([a_i, b_i]\) 表示第 \(i\) 区间, 要求合并所有有重叠的区间 (断点处相交也算交集), 返回一个不重叠的区间数组。该数组需恰好覆盖输入中的所有区间.

    思路#

    1. 排序: 按照区间左端点从小到大排序;
    2. 贪心求解: 维护一个合并后区间的尾端点 \(tail\), 当遍历到第 \(i\) 区间 \([a_i, b_i]\),
      • 如果 \(a_i <= tail\), 说明区间 \([a_i, b_i]\) 与前面的区间有交集, 无需增加区间个数, 且 \(tail = \max(tail, b_i)\);
      • 如果 \(a_i > tail\), 说明区间 \([a_i, b_i]\) 与前面的区间无交集, 需增加区间个数, 且 \(tail=b_i\).

    相关题目: Leetcode 56. 合并区间

    无重叠区间#

    题意#

    给定一组区间的集合, 要求选择尽量多的区间，使得这些区间互不相交，求可选取的区间的最大数量, 其中端点相同也算有重复。

    思路#

    1. 排序: 按照区间 右端点 从小到大排序;
    2. 贪心求解: 维护一个前序区间的尾端点 \(tail\), 从前往后依次遍历每个区间, 对于第 \(i\) 个区间 \([a_i, b_i]\),
      • 如果 \(a_i <= tail\), 说明 区间 \([a_i, b_i]\) 与前面的区间有交集, 直接跳过；
      • 如果 \(a_i > tail\), 说明区间 \([a_i, b_i]\), 选中当前区间, 并设置 \(tail = b_i\).

    相关题目: LeetCode 435. 无重叠区间

    区间选点问题#

    题意#

    给定一组区间的集合, 取 尽量少 的点, 使得每个区间内 至少有一个点 (不同区间内可以包含相同的点，区间断点处的点也算作区间内).

    思路#

    方法一: 可转化为 无重叠区间问题, 讲点的位置选在重合处即可。

    方法二:

    1. 排序: 按照区间 右端点 从小到大排序;
    2. 贪心求解: 维护一个前序区间的尾端点 \(tail\), 从前往后依次遍历每个区间, 对于第 \(i\) 个区间 \([a_i, b_i]\),
      • 如果 \(a_i <= tail\), 说明 区间 \([a_i, b_i]\) 与前面的区间有交集, 前面已经选点, 直接跳过；
      • 如果 \(a_i > tail\), 说明区间 \([a_i, b_i]\), 选中当前区间中的点, 并设置 \(tail = b_i\).

    相关题目: LeetCode 452. 用最少数量的箭引爆气球

    区间覆盖#

    题目#

    给定一组区间的集合和一个目标区间, 选择可覆盖目标区间的最少区间数量。

    思路#

    1. 排序: 按照区间 左端点 从小到大排序;
    2. 贪心求解: 从前往后依次遍历每个区间,
      • 选择所有能覆盖当前目标区间起始位置 \(start\) 的区间的集合 \(S=\left\{[a_i, b_i], [a_{i+1}, b_{i+1}], \cdots, [a_{j}, b_{j}]\right\}\);
      • 选择 \(S\) 中右端点最大的区间. 若右端点最大的区间为第 \(k\) 个区间, 其右端点为 \(b_k\), 更新 \(start=b_k\).

    相关题目: LeetCode 1024. 视频拼接

    区间分组#

    题意#

    给定一组区间的集合, 问最少可以将这些区间分成多少组, 使得每个组内的区间互不相交。

    思路#

    1. 排序: 按照区间 左端点 从小到大排序;
    2. 贪心求解: 假定现在已经分成 \(m\) 组, 第 \(k\) 组的最右端点是 \(r_k\), 当前区间的范围为 \([a_i, b_i]\), 从前往后依次遍历每个区间。
      • 如果 \(a_i \leq r_k\), 第 \(i\) 个区间不能加入第 \(k\) 组;
      • 如果 \(a_i > r_k\), 第 \(i\) 个区间可以加入第 \(k\) 组;
      • 若 \(m\) 组中没有组可接受当前区间, 则新开一个组, 加入区间 \(i\);
      • 如存在可接受当前区间的组 \(k\), 加入当前区间后更新 \(r_k=b_i\).

    实现: 为了快速找到接受当前区间的组, 可以使用 优先队列 (小顶堆), 优先队列记录每个组右端点的值，每次在 \(O(1)\) 时间拿到右端点最小值。

    相关题目: NC147 主持人调度（二）

    class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        # 按左端点排序
        intervals.sort()

        results = [intervals[0]]
        tail = intervals[0][1]
        for a, b in intervals[1:]:
            if a <= tail:
                # 有交集, 合并
                tail = max(tail, b)
                results[-1][1] = tail
            else:
                # 无交集, 新增
                tail = b
                results.append([a, b])
        return results

    57.插入区间#

    题目: 给你一个 无重叠的，按照区间起始端点排序的区间列表。 在列表中插入一个新的区间，你需要确保列表中的区间仍然有序且不重叠（如果有必要的话，可以合并区间）。

    方法一: 两步走思路#

    该思路比较直接, 没那么多弯弯绕绕。

    1. 将 newInterval 按序插入 intervals;
    2. 合并重叠区间, 如 合并区间.

    方法二: 区间分析法#

    两个区间 \(S=[s_l, s_r]\) 和 \(T=[t_l, t_r]\) 的关系:

    • 无交集:
      • \(S\) 在 \(T\) 左侧, \(s_r < t_l\);
      • \(S\) 在 \(T\) 右侧, \(t_r < s_l\).
    • 有重叠: 不满足无交集的情况则必定有交集, 有
      • 交集: \([\max(s_l, t_l), \min(s_r, t_r)]\);
      • 交集: \([\min(s_l, t_l), \max(s_r, t_r)]\).

    基于以上分析, 对于 intervals 中无交集的区间可直接加入结果列表中, 其关键是如何处理有交集的区间。定义 intervals 第 i 个区间为 \([l_i, r_i]\), newInterval 表示的区间为 \([left, right]\), 则

    • 无交集: 如果 \(r_i < left\) 或者 \(l_i > right\), 直接将 \([l_i, r_i]\) 加入答案；
    • 有交集: 需合并区间, 无需将 \([l_i, r_i]\) 加入答案。

    class Solution:
    def insert(self, intervals: List[List[int]], newInterval: List[int]) -> List[List[int]]:
        if not intervals:
            return [newInterval]

        # 第1步: 将 newInterval 插入 intervals 中, TODO: 二分查找法优化
        inserted = False
        for i, interval in enumerate(intervals):
            if newInterval[0] < interval[0]:
            intervals.insert(i, newInterval)
            inserted = True
            break
        if not inserted:                   # 未插入则 append
            intervals.append(newInterval)

        # 第2步: 合并有序区间
        results = [intervals[0]]
        for interval in intervals[1:]:
            if interval[0] <= results[-1][1]:
            # 有交集, 合并
            results[-1][1] = max(results[-1][1], interval[1])
            else:
            # 无交集, 新增
            results.append(interval)
        return results

    NC147 主持人调度（二）#

    【描述】

    有 n 个活动即将举办，每个活动都有开始时间与活动的结束时间，第 i 个活动的开始时间是 \(start_i\) ,第 i 个活动的结束时间是 \(end_i\), 举办某个活动就需要为该活动准备一个活动主持人。

    一位活动主持人在同一时间只能参与一个活动。并且活动主持人需要全程参与活动，换句话说，一个主持人参与了第 i 个活动，那么该主持人在 \((start_i,end_i)\) 这个时间段不能参与其他任何活动。求为了成功举办这 n 个活动，最少需要多少名主持人。

    复杂度要求：时间复杂度 \(O(n\log n)\)，空间复杂度 \(O(n)\).

    【示例】

2
3
    输入: 2,[[1,2],[2,3]]
返回值: 1
说明: 只需要一个主持人就能成功举办这两个活动

    输入：2,[[1,3],[2,4]]
返回值：2
说明：需要两个主持人才能成功举办这两个活动
