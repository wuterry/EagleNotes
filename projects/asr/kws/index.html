    项目需求与挑战#

    需求:构建一个 可见即可说，实时、离线、多语言支持的命令词识别系统。

    挑战:

    • 数据搜集困难: (1) 老外语音采集困难; (2) 无大规模命令词公开数据集可用;
    • W5100 算力弱: 只相当于 IPhone16 Pro 1/1700;
    • 开源方案效果差: sherpa-onnx 采用 ASR 方案，mic 的英文识别率都不到 50%；
    • 三方基于 Mic SDK 无法抵抗高风噪。

    前端特征#

    基本流程:

    graph LR;
预加重-->分帧 -->加窗-->FFT-->FBank-->MFCC

    预加重#

    语音信号常伴有频谱倾斜（Spectral Tile）现象，即高频部分幅度较低频部分幅度小，预加重通过一阶滤波增强高频部分，达到平衡频谱的目的。

    \[ y(t) = x(t) - \alpha x(t-1), 0.95 < \alpha < 0.99 \]

    分帧#

    • 原因：语音信号的频率会随时间变化，而信号处理算法（如FFT）需要信号是稳定的；此外大部分场景下语音识别需要实时识别。
    • 假设：假定很短的时间内信号的频率是稳定的，不会发生很大变化；

    分帧即将信号分割成短时帧，通常为 20~40ms，相邻帧有一定程度的交叠。在我们的算法中，帧长为 32ms, 帧移为 10ms.

    加窗#

    对每一帧信号加窗, 使每帧两端平滑衰减, 降低 FFT 后的旁瓣强度, 改善频谱质量。常见的窗如: 矩形窗，Hamming 窗，Hanning 窗。Hamming 窗函数如下:

    \[ w(n) = 0.54 - 0.46 * \cos\left(\frac{2\pi n}{N-1}\right), 0 \le n \le N - 1 \]

    FFT#

    对每一帧的加窗后的信号进行 N 点 FFT 变换（STFT）, N通常取 256, 512或1024, 并计算能量谱:

    \[ X(k)=\sum_{j=1}^{N} x(j) \omega_N ^ {(j-1)(k-1)} \]
    \[ P=\frac{|FFT(x_i)|^2}{N} \]

    为什么只使用能量谱？相位谱有什么用?

    1. 人人对相位的敏感性低；能量谱能有效捕捉声道特征; 只使用能量谱能提升计算效率；
    2. 相位谱使用场景：音频合成、音频增强 (如消噪和回声消除)、音乐信号处理。

    FBank (Filter Bank) 特征#

    在经过 FFT, 在 能量谱 上应用 Mel 滤波器组便能提取到 FBank 特征。

    人耳在接受声音时呈现非线性状态，对高频部分更不敏感。 而 Mel 刻度是一个能模拟人耳接收声音规律的刻度，在低频区分辨度更高，高频区分辨度更低，换算关系如下:

    \[ m = 2595 * \log_{10} \left(1 + \frac{f}{700} \right) \]

    Mel 滤波器组是一系列三角形滤波器，通常为 40个或者80个, 对于每一个滤波器在中心频率响应值为1，两边线性衰减到 0.

    MFCC#

    FBank 特征往往高度相关。可进一步使用 DCT 变换将这些相关的滤波器系数进行压缩。ASR 中通常取 2~13维，通常还会对 MFCC 做正弦提升 (Sinusoidal Liftering)，可以提升在噪声信号的识别率。

    \[ MFCC_i^{'}=\omega_i MFCC_i, 其中 \omega_i = \frac{D}{2} \sin \left( \frac{\pi * i}{D} \right) \]

    详细代码见 asr_fbank_mfcc.ipynb

    DKD Loss 1#

    动机: 1. SOTA 算法主要集中在如何蒸馏特征层，而忽略了 logits 层。 2. 将经典的 KD_loss 拆分为两部分，分别是 目标类别知识蒸馏 (TCKD) 和 非目标类别知识蒸馏 (NCKD). TCKD 传授给 "学生" 的知识是当前训练样本的难度，而 NCKD 才是知识蒸馏有效的主要原因。常规的 KD 方法会抑制 NCKD 的作用，并且限制了这两部分的灵活度。因此需要解耦。

pt_stu, pnt_stu = prob_stu[t], prob_stu[1-t].sum(1)
pt_tea, pnt_tea = prob_tea[t], prob_tea[1-t].sum(1)

# TCKD
tckd = kl_div(log(pt_stu), pt_tea) + kl_div(log(pnt_stu), pnt_tea)
# NCKD

# DKD
dkd_loss = (alpha*tckd + beta*nckd) * T**2

    动机#
    • 基于熵最大化理论，推导包含温度系数的 Softmax 函数中，温度没有明显的约束条件，亦即没有理论强制学生和教师网络全局共享温度系数；
    • 已有“温度共享模式”导致两个问题：（1）学生网络被迫输出与教师网络相当的 logits;（2）KL 难以真实反映蒸馏效果。

    传统知识蒸馏默认学生/教师网络的温度是全局一致的，这种设置迫使学生模仿教师的logit的具体值，而非其关系，论文方法提出logit标准化改善这个问题。
